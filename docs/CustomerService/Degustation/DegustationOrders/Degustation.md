# Внутренний заказ

Документ **«Внутренний заказ»** отражает потребность на отгрузку товаров на отличные от заказов клиентов и внутренних перемещений операции (например, дегустация, реклама и т.д.).  
Документы расположены в разделе **"Заказы"** - **"Внутренние заказы"**.  

При создании документа автоматически подставляются статус документа и этап (если используются в системе).

На вкладке **"Основное"** указываются:

- **Номер документа** (заполняется автоматически)
- **Дата** (дата создания документа в системе)
- **Организация** (видна, если в системе несколько организаций. Заполняется автоматически, если указана основная организация)
- [Цель](../Target/Target.md)
- **Инициатор** (Пользователь или Физ лицо)
- Подразделение (заполняется автоматически, если указано основное подразделение)
- **Получатель** (Точка доставки или Физ лицо)
- Адрес доставки заполняется из точки доставки, можно заполнить вручную
- **Тип доставки** Самовывоз, Доставка сторонней транспортной компанией, Доставка нашей транспортной компанией. Заполняется из точки доставки, можно заполнить вручную  
- **Склад отгрузки** заполняется из точки доставки, можно заполнить вручную (виден, если в системе заказ возможен только с одного склада, иначе указывается на вкладке Товары)  
- Информация о дате отгрузки и дате и времени доставки.  
Даты рассчитываются автоматически при заполнении одной из них с учетом количества дней доставки, указанного в точке доставки.   
Время доставки заполняется из точки доставки, можно заполнить вручную. 

![1]

**Вкладка "Товары"**

На вкладке Товары указывется список заказываемой номенклатуры.  
Заполняются номенклатура, её характеристика, количество упаковок, упаковка (колонка Ед.).   

По кнопке "Заполнить" - "Склад отгрузки" для выделенных строк можно заполнить склад отгрузки.  
*Кнопка доступна, когда в системе есть возможность создавать один заказ с разных складов отгрузки.* 

!!! attention ""  

    Обязательно к заполнению поле «Статья затрат» (заполняется по цели или вручную из справочника Статьи затрат), можно указать аналитику затрат. Это нужно для дальнейшего анализа внутренних потреблений.  

**Вкладка "Дополнительно"**  

На вкладке Дополнительно автоматически заполняется способ создания заказа и ответственный (пользователь, который создал заказ в системе).

# Формирование отгрузок

Далее заказ может [отгружаться](../../FormationOfShipments/FormationOfShipmentsScheme.md) по обычному пути заказа клиента. 

Конечным документом, отражающим факт отгрузки товара, является документ **«Внутреннее потребление»**
Документы расположены в разделе **"Заказы"** - **"Внутреннее потребление"**.  

На вкладке **"Основное"** указываются:

- **Номер документа** (заполняется автоматически)
- **Дата** (дата создания документа в системе)
- **Организация** (видна, если в системе несколько организаций. Заполняется автоматически, если указана основная организация)
- Основание (Внутренний заказ) заполняется автоматически, если документ создан на основании заказа
- [Цель](../Target/Target.md)
- **Инициатор** (Пользователь или Физ лицо)
- Подразделение (заполняется автоматически, если указано основное подразделение)
- **Получатель** (Точка доставки или Физ лицо)
- Адрес доставки заполняется из точки доставки, можно заполнить вручную
- Задание на доставку заполняется автоматически, если документ внутренний заказ требовал доставки

**Вкладка "Товары"**

На вкладке Товары указывется список заказываемой номенклатуры.  
Заполняются номенклатура, её характеристика, количество упаковок, упаковка (колонка Ед.).   

По кнопке "Заполнить" - "Склад отгрузки" для выделенных строк можно заполнить склад отгрузки.  

!!! attention ""  

    Обязательно к заполнению поле «Статья затрат» (заполняется по цели или вручную из справочника Статьи затрат), можно указать аналитику затрат. Это нужно для дальнейшего анализа внутренних потреблений.  

**Вкладка "Дополнительно"**  

На вкладке «Дополнительно» указываются способ создания заказа, ответственный (пользователь, который создал заказ в системе), вид цен внутреннего потребления (заполняется из константы, но можно изменить руками).   

!!! attention ""  

    Документ «Внутреннее потребление» можно скорректировать единственным образом – корректировкой первичной накладной.   

# Настройки исполнения заказов

В разделе **Заказы** в подсистеме **"Сервис"** - **"Настройки исполнения заказов"** можно включить дополнительные настройки создания заказов.

**"Контроль остатков заказов"** не позволяет после оформления по заказу плана отгрузки сделать количество товаров в заказе клиента меньше чем в плане.

При этом количество в плане отгрузки нельзя сделать больше чем в заказе.

!!! attention ""  

    Настройка распространяется на все виды заказов. 

**"Заказывать с нескольких складов"** разрешает создавать в системе заказы с отгрузкой с нескольких складов. В таком случае склад отгрузки указывается на вкладке Товары для каждой строки номенклатуры.

!!! attention ""  

    Настройка распространяется на заказы клиентов и заказы на внутреннее потребление.

Константа **"Вид цен для внутреннего потребления"** используется для автоматического заполнения вида цен в документах Внутреннее потрбление.
